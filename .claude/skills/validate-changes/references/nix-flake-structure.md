# Flake structure

This flake builds and packages QML shell components for Quickshell, and exports a home-manager module for consumers.

## Top-level outputs

```
packages.x86_64-linux
├── kh-launcher          — app launcher config derivation (shell.qml + NixConfig/NixBins)
├── kh-cliphist          — clipboard history config derivation
└── cliphistDecodeAll    — standalone cliphist decode-all shell script

homeManagerModules
└── default              — home-manager module; defines programs.kh-ui options

checks.x86_64-linux
└── tests                — QML unit test suite (qmltestrunner)

apps.x86_64-linux
├── kh-launcher          — run kh-launcher directly (hardcoded Catppuccin colors)
├── kh-cliphist          — run kh-cliphist directly
└── screenshot           — headless screenshot tool
```

## Attribute path patterns

**Packages:**
```
.#packages.x86_64-linux.<name>
```

**home-manager module options** (evaluated via the consuming flake at ~/nix):
```
/home/kennethhoff/nix#nixosConfigurations.<host>.config.home-manager.users.kennethhoff.programs.kh-ui
```

## Source layout

```
flake.nix                 — flake outputs (top-level)
src/                      — all source
  apps/                   — all app source
    kh-bar.qml            — bar app entry point
    kh-cliphist.qml       — clipboard history app entry point
    kh-launcher.qml       — launcher app entry point
    kh-view.qml           — file/image viewer app entry point
    bar/                  — bar infrastructure + plugins
      plugins/            — built-in bar plugins (Clock, Volume, Workspaces, …)
      BarPlugin.qml       — plugin base class
      BarRow.qml, …       — layout and panel infrastructure
    cliphist/             — cliphist-specific components
    launcher/             — launcher-specific components
  lib/                    — cross-app reusable QML components
  scripts/                — Nix helper scripts (cliphist-decode-all.nix)
  themes/                 — colour palette presets
  config.nix              — generates NixConfig.qml from colors + font settings
  ffi.nix                 — generates NixBins.qml from store-path binaries
  bar-config.nix          — generates per-instance BarLayout files
  hm-module.nix           — home-manager module (curried over self)
  stylix-integration.nix  — Stylix → kh-ui theme bridge
tests/                    — qmltestrunner test files (tst_*.qml)
docs/                     — reference docs and feature backlog
```

## How configs are built

Each app config is a derivation assembled by `runCommandLocal`:
```
$out/
  shell.qml      ← from src/apps/<app>.qml
  NixConfig.qml  ← generated by src/config.nix with colors + font settings
  NixBins.qml    ← generated by src/ffi.nix with absolute store-path binaries
  lib/
    *.qml        ← from src/lib/
```

The pre-built packages (`kh-launcher`, `kh-cliphist`) use hardcoded Catppuccin Mocha colors. The home-manager module builds the same structure at eval time using Stylix colors from the consumer's config.
